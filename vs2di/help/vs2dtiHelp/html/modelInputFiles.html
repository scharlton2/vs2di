VS2DT 3.3 Model Input Files

Model Input Files contain ASCII data that are read by the numerical model VS2DT 3.3.

When you have entered all required data in the preprocessor, and you invoke the postprocessor (by selecting "Postprocessor" from the "Show" menu), two model input files (vs2dh.fil and vs2dh.dat) are generated in the working directory (see ). This is done automatically and requires no handling by the user.

Therefore, in most cases, users may ignore the model input files.

The information below is provided for knowledgeable users who want to generate model input files without invoking the postprocessor. This is known as exporting, and can be done as follows:

1. Make sure all required simulation setup data are entered in the preprocessor.
2. Click the "File" menu, and select "Export data". The Export Data dialog box is displayed.
3. Navigate to the desired folder and enter the name of the namelist file (see below). By convention, this file should end with the extension ".fil"--this convention is recommended but is not required.
4. Click "OK"

The export process generates two files in the specified folder:

• The namelist file, as specified in step 3 above, contains the names of input and output files to be opened by VS2DT 3.3.
• The model data file, which is always named vs2dt.dat, contains the input data to VS2DT 3.3.

Exporting provides the flexibility for knowledgeable users to manually alter the model data file (using a text editor). Such modification is necessary if the preprocessor is not able to create a model setup in a particular configuration. The modified model data can be run by launching the standalone postprocessor.
