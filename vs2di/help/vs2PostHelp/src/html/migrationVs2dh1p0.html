Migration from VS2DH 1.0

Note: This feature is intended for knowledgeable users who want to work with existing data files originally prepared for VS2DH Version 1.0. The migration procedure needs to be applied only once.

The user may apply the following migration procedure:
• Partial migration. This procedure can be quickly applied and does not require modifying the model data file. However, inconsistencies between VS2DH 1.0 and 3.3 data definitions are maintained. These inconsistencies are automatically resolved in the model code. However, this feature might not be maintained in future versions.

After migration, users may to run a simulation by loading the namelist file to the standalone postprocessor.
