Migration from VS2DT 2.5

Note: This feature is intended for knowledgeable users who want to work with existing data files originally prepared for VS2DT 2.5. The migration procedure needs to be applied only once.

The user may apply one of the following migration procedures:
• Partial migration. This procedure can be quickly applied and does not require modifying the model data file. However, inconsistencies between VS2DT 2.5 and 3.3 data definitions are maintained. These inconsistencies are automatically resolved in the model code. However, this feature might not be maintained in future versions.
• Full migration. This procedure may be more time consuming than partial migration, and requires modifying the model data file. However, after full migration, the data file is fully consistent with VS2DT 3.3.

After migration, users may to run a simulation by loading the namelist file to the standalone postprocessor.
